## Introduction

The course aims to build essential skills in Secure Software Concepts, Requirements, Design, Implementation/Coding, and Testing. It highlights the security practices that must be integrated across the software development lifecycle. The course offers a thorough overview, addressing the key aspects of people, processes, and technology related to software security defenses.  In addition, this course intends to cultivate a positive, ethical, and responsible mindset in vulnerability management, from hunting to reporting.

## Course Outline

| **Week** | **Topics Covered**                                                                                 |
|----------|----------------------------------------------------------------------------------------------------|
| **Week 1**  | - Introduction to Secure Software Concepts  <br> - Importance of secure software design  <br> - Overview of secure software principles  <br> - System issues and properties in secure software |
| **Week 2**  | - Security in the Software Development Life Cycle (SDLC)  <br> - Security concepts integrated into SDLC phases  <br> - Secure software requirements: sources and types |
| **Week 3**  | - Risk Management in Software Projects  <br> - Identifying, analyzing, and mitigating risks  <br> - Introduction to security standards (e.g., NIST, FIPS) |
| **Week 4**  | - Security principles and secure design considerations  <br> - Designing secure design principles  <br> - Design processes and best practices (e.g., OWASP guides)  <br> - Security methodologies (e.g., STRIDE, DREAD, OCTAVE) |
| **Week 5**  | - Secure Software Frameworks  <br> - Introduction to frameworks (e.g., Zachman, COBIT, SABSA)  <br> - Practical examples of applying frameworks |
| **Week 6**  | - Security Mechanism  <br> - Mechanism and Policy  <br> - How to regulate access  <br> - The K.I.S.S Principle |
| **Week 7**  | - Common Software Vulnerabilities  <br> - Overview of vulnerabilities (e.g., buffer overflow, SQL injection)  <br> - Exploitation techniques and impacts |
| **Week 8**  | - Defensive Coding Practices  <br> - Concepts and techniques for defensive coding  <br> - Avoiding vulnerabilities and polymorphic malware attacks |
| **Week 9**  | - Secure Software Implementation and Coding  <br> - Best practices for secure coding  <br> - Software development methodologies |
| **Week 10** | - Secure Software Testing  <br> - Testing concepts: functional, non-functional, and security testing  <br> - Security testing methodologies and tools  <br> - Static and Dynamic analysis |
| **Week 11** | - Software Security Testing and Acceptance  <br> - Software acceptance criteria from a security perspective  <br> - Testing against common attack vectors |
| **Week 12** | - Authentication Protocols  <br> - Common Pitfalls  <br> - Ways to Analyze Protocols  <br> - Login-only protocols  <br> - Mutual authentication with Key Distribution Center |
| **Week 13** | - Secure Modern Web Development  <br> - Browser Security Mechanism  <br> - Building blocks for secure modern web applications  <br> - The future of user authentication on the web |
| **Week 14** | - Web security in the real world  <br> - Client-side attacks and defenses  <br> - Server-side attacks and defenses |
| **Week 15** | - Emerging trends in secure software engineering (e.g., AI security)  <br> - Recap of major concepts covered  <br> - Final Q&A and discussions |
